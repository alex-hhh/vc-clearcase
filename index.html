vc-clearcase project homepage

This is the homepage for the vc-clearcase project, which provides integration with ClearCase for GNU Emacs. Please visit the SourceForge.net homepage for the project here.

vc-clearcase.el

The package supports full VC functionality and also adds some ClearCase specific functionality like updating static views, edit a view's configspec, list checkouts and start the version tree browser GUI on a file. Once the package is loaded, these additional commands will show up in the Tools/Version Control menu.

Highlights of the package:

  • Version control operations are very responsive. The package communicates with the cleartool process using a pipe and uses various caching techniques to improve responsiveness.
  • Being a back-end to vc.el, all packages that use vc.el work with ClearCase files (for example the ediff package and add-log).
  • The annotate mode allows navigating the versions of an annotated file and diff-ing these versions (vc.el from CVS GNU Emacs is required for this) in addition to the nice highlighting that vc.el provides.
  • The package provides an easy to use interface to cleartool (see the ah-cleartool/ah-cleartool-do functions). This allows implementing complex operations on ClearCase files in Emacs-Lisp (see vc-clearcase-label-diff-report for an example).

Installation

The package comes in a single file, which you can download here.
  1. Put this file in the site-lisp directory of your GNU Emacs installation and byte-compile it.
  2. Open open vc-clearcase.el in Emacs, than execute the code below. This will create a file named vc-clearcase-auto.el containing the autoloads for the file. 
    (with-current-buffer (get-buffer "vc-clearcase.el")
  (let* ((file (buffer-file-name (current-buffer)))
         (dir (file-name-directory file))
         (generated-autoload-file
          (expand-file-name "vc-clearcase-auto.el" dir)))
    (update-file-autoloads file)))
  3. Add the following line to your ~/.emacs.el file: 
    (load "vc-clearcase-auto")
If you don't want to generate autoloads, you will have to add the symbol CLEARCASE (in uppercase) to `vc-handled-backends' via customize, to tell the vc.el package about the new back end (the autoloads above do that automatically for you.)
